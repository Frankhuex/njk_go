# Go 迁移开发计划

## 目标

- 将 Python WebSocket 接口能力逐步迁移到 Go
- 保持现有群聊收发、命令处理、数据库落库、外部服务调用的整体行为一致
- analyzer 报告删除主题提取与分词统计，只保留消息量、活跃日期、熬夜之巅、被 @ 最多等统计

## 约束

- 不删除仓库
- 不切换或删除分支
- 不修改 Python 项目代码
- 遇到阻塞先绕过，优先推进其他可落地模块

## 当前现状

- Go 已具备 NapCat 入站类型、解析器、WebSocket 服务、数据库接线、命令分发、图片消重、报告、BBH 与 AI 调用骨架
- Go 服务已可启动并已观察到 NapCat 自动连接 11004
- Python 仍作为行为对照来源，核心入口在 `NJK/src/main.py`
